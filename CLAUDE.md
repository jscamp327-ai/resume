# 프로젝트 지침 — resume (이력서·포트폴리오)

> 본 레포는 **이력서 및 포트폴리오 관리 전용 저장소**이다.
> 한국어·영어 이력서와 HTML 포트폴리오를 함께 관리한다.

---

## 0. 일반 규칙

- **응답 언어**: 모든 응답은 **한국어**로 작성한다 (코드·기술 용어 제외)

---

## 1. 파일 구조 및 역할

| 파일/폴더 | 역할 |
|----------|------|
| `index.html` | **한국어 이력서 원본 (편집 주파일)** 겸 GitHub Pages 진입점 |
| `resume_en.html` | **영어 이력서 원본 (편집 주파일)** |
| `portfolio.html` | 포트폴리오 페이지 (HTML) |
| `portfolio_assets/` | 포트폴리오 이미지·리소스 |
| `MyFileTemp/bak/` | 과거 DOCX·PDF 아카이브 (직접 수정 금지, 참고용) |

---

## 2. 작업 규칙

### 원본 관리
- 이력서 원본은 **HTML 기준**으로 관리한다 (`index.html`, `resume_en.html`)
- DOCX·PDF는 더 이상 원본이 아니며, `MyFileTemp/bak/`에 아카이브로만 보관한다
- 버전 관리는 git 이력으로 대체한다 (별도 `_bak` 파일 불필요)

### 한국어·영어 이력서 동기화
- 경력·기술스택 등 주요 내용 변경 시 한국어·영어 이력서 모두 반영한다
- `index.html`(한국어)과 `resume_en.html`(영어)은 내용이 동기화된 상태를 유지한다

### 문서 출력 (PDF · Word)
- 필요 시 HTML(`index.html`, `resume_en.html`)을 기준으로 PDF·Word 파일을 출력한다
- 출력된 PDF·Word 파일은 산출물이므로 직접 편집하지 않는다

---

## 3. 콘텐츠 가이드라인

### 한국어 이력서
- 문체: 명사형 마무리 (예: "~개발", "~구축", "~운영")
- 수치·성과를 가능한 한 포함한다 (예: "응답속도 30% 개선")

### 영어 이력서
- 문체: 과거형 액션 동사 시작 (예: "Developed", "Optimized", "Led")
- 수치·성과를 함께 기술한다

### 포트폴리오 (`portfolio.html`)
- 프로젝트별 섹션으로 구성한다
- 이미지 리소스는 `portfolio_assets/`에 보관한다

---

## 5. 배포

- GitHub Pages를 통해 배포한다 (`index.html`이 진입점)
- 배포 전 `index.html`, `portfolio.html`, `resume_en.html` 로컬 확인 후 push한다
